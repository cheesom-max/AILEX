# AILEX 커뮤니티 + PLG (Product-Led Growth) 전략

> 작성일: 2026-02-14
> 목표: 커뮤니티를 통한 오가닉 성장 + 제품 자체가 성장 엔진이 되는 PLG 구조 설계

## Part 1: 커뮤니티 전략

### 1.1 자체 커뮤니티 vs 기존 커뮤니티 활용

| 기준 | 자체 커뮤니티 | 기존 커뮤니티 활용 |
|------|-------------|------------------|
| 초기 비용 | 낮음 (무료 도구 활용) | 없음 |
| 구축 시간 | 3~6개월 | 즉시 시작 |
| 통제력 | 높음 | 낮음 (플랫폼 규칙 따름) |
| 신뢰도 | 시간이 걸림 | 기존 커뮤니티 신뢰 활용 |
| 추천 | Pre-PMF: 기존 커뮤니티 우선 | Post-PMF: 자체 커뮤니티 구축 |

**전략**: Pre-Launch ~ Pre-PMF 단계에서는 기존 커뮤니티 활용에 80%의 에너지를 집중하고, 자체 커뮤니티는 카카오톡 오픈채팅부터 가볍게 시작합니다.

### 1.2 기존 커뮤니티 활용 전략

#### 커뮤니티 1: LinkedIn (최우선)

**활동 목표**: AI 기본법 컴플라이언스 분야의 "Go-To Person"이 되기

**주간 활동 계획**:
| 요일 | 활동 | 시간 |
|------|------|------|
| 월 | 전문 인사이트 포스트 발행 (AI 기본법 해설) | 30분 |
| 화 | 타인 포스트에 가치 있는 댓글 10개 | 20분 |
| 수 | 카루셀/인포그래픽 포스트 발행 | 30분 |
| 목 | DM 아웃리치 5건 + 댓글 응대 | 30분 |
| 금 | 주간 AI 기본법 뉴스 요약 포스트 | 20분 |

**콘텐츠 유형**:
1. **법 조문 해설 시리즈**: "AI 기본법 제27조 3분 요약" (주 1회)
2. **실무 가이드**: "고영향 AI 자가진단 체크리스트" (격주 1회)
3. **업계 뉴스 코멘트**: AI 기본법 관련 뉴스에 전문가 의견 추가 (수시)
4. **빌딩 인 퍼블릭**: "AILEX 개발일지 #N" (주 1회)
5. **데이터/인사이트**: "자가진단 결과 통계: N%가 고영향 AI 해당" (월 1회)

**팔로워 성장 목표**: 0 -> 500(30일) -> 2,000(90일)

#### 커뮤니티 2: 디스콰이엇 (disquiet.tech)

**활동 계획**:
- 프로덕트 등록: "AILEX - AI 기본법 컴플라이언스 자동화"
- 메이커 로그 주 1회: 개발 과정, 기술 선택, 사용자 피드백 반영
- 다른 메이커의 프로덕트에 건설적 피드백 제공 (주 3건)
- 런칭 시 커뮤니티 투표 요청

**기대 효과**: 런칭 주 300~500 방문, 50~100 가입

#### 커뮤니티 3: GeekNews (news.hada.io)

**활동 계획**:
- "Show" 게시물: "AI 기본법 고영향 AI 자동 판정 도구를 만들었습니다"
- 기술 스택 + 프롬프트 엔지니어링 경험 공유
- AI 기본법의 기술적 함의 분석 글

**기대 효과**: 게시물 당 1,000~5,000 조회, 100~300 클릭

#### 커뮤니티 4: 카카오톡 오픈채팅 / 슬랙

**활용 방법**:
- 기존 AI 스타트업 오픈채팅방에 참여 (방장에게 사전 양해)
- AI 기본법 관련 질문에 적극 답변 (홍보 아닌 도움 중심)
- 자체 "AI 기본법 대응 커뮤니티" 오픈채팅 운영 (30명부터 시작)

#### 커뮤니티 5: AI 기본법 관련 오프라인 행사

**활동 계획**:
- 한국인공지능협회/산업협회 세미나 참석 (월 1~2회)
- 참석자 명함 수집 + 후속 이메일/DM
- 자체 워크숍 개최 (월 1회, 온오프믹스/이벤터스)

### 1.3 자체 커뮤니티 구축 (0 -> 100 -> 1,000명)

#### Phase 1: 시드 (0 -> 30명, 1~2주)

**플랫폼**: 카카오톡 오픈채팅방

**이유**: 한국 B2B에서 카카오톡은 가장 접근성 높은 채널. 별도 앱 설치 불필요.

**시드 멤버 확보**:
1. 베타 테스터 10~20명을 오픈채팅에 초대
2. 콜드 이메일 응답자 중 관심 표현한 사람 초대
3. LinkedIn 연결 중 AI 기본법에 관심 있는 사람 10명 초대

**운영 규칙**:
- 주제: AI 기본법 대응 실무 정보 교환
- 금지: 홍보/광고, 정치, 종교
- 운영자: 주 3회 이상 AI 기본법 관련 유용한 정보 공유

#### Phase 2: 성장 (30 -> 100명, 1~2개월)

**액션**:
1. 오픈채팅방에서 매주 "AI 기본법 Q&A 타임" 운영 (수요일 오후 3시, 30분)
2. 멤버 전용 "AI 기본법 최신 뉴스" 주간 요약 공유
3. 세미나/워크숍 참석자를 오픈채팅으로 유도
4. 디스콰이엇/GeekNews에서 오픈채팅방 홍보

**전환 포인트**: 100명 이상 되면 카카오톡 한계 (검색/아카이빙 불편) -> Slack/Discord로 이전 검토

#### Phase 3: 확장 (100 -> 1,000명, 3~6개월)

**플랫폼 전환**: Slack 워크스페이스 (B2B 표준)

**Slack 채널 구조**:
```
#general          - 일반 대화, 공지
#ai-기본법-뉴스    - AI 기본법 관련 뉴스/법령 변경
#자가진단-후기     - AILEX 자가진단 결과/경험 공유
#영향평가서-팁     - 영향평가서 작성 실무 팁
#고영향ai-사례     - 고영향 AI 판정 사례 공유
#질문-답변         - AI 기본법 관련 질문
#채용-파트너       - 컴플라이언스 관련 채용/협업
#ailex-피드백      - AILEX 제품 피드백/기능 요청
#오프라인-모임     - 밋업/세미나 정보
```

**성장 플레이북**:
1. AILEX 가입 시 "커뮤니티 참여하기" CTA (온보딩 이메일에 포함)
2. 매주 수요일 "AI 기본법 라이브 Q&A" (Slack Huddle, 30분)
3. 월 1회 "AI 기본법 실무 웨비나" 개최 (Zoom, 커뮤니티 멤버 우선 참여)
4. 활발한 멤버에게 "AILEX Community Champion" 배지 부여
5. 커뮤니티 멤버 전용 혜택: Pro 플랜 첫 달 20% 할인

## Part 2: PLG (Product-Led Growth) 전략

### 2.1 PLG 기본 구조

AILEX의 PLG는 **무료 자가진단이 핵심 성장 엔진**입니다.

```
[무료 자가진단] -- PLG Entry Point
       ↓
  5분 만에 결과 확인 = Aha Moment
       ↓
  "고영향 AI 해당" 판정 = 구매 동기 생성
       ↓
  "영향평가서를 생성하세요" CTA = 업그레이드 트리거
       ↓
  [Pro 플랜 전환] = Revenue
       ↓
  컴플라이언스 완료 + 배지 = 바이럴 확산
```

### 2.2 무료 -> 유료 전환 포인트

#### 전환 포인트 1: 자가진단 결과 확인 직후 (가장 강력)

```
[자가진단 결과: "고영향 AI 해당"]
        ↓
"귀사의 AI 서비스는 고영향 AI에 해당합니다.
 다음 의무사항을 이행해야 합니다:
 1. 영향평가서 작성
 2. 투명성 보고서 작성
 3. 위험관리계획서 수립

 지금 Pro 플랜에서 AI가 30분 만에 모든 문서를 생성합니다.
 [Pro 플랜으로 문서 생성하기] [파운더 플랜 50% 할인]"
```

**전환 심리**: "고영향 AI에 해당한다"는 판정이 긴급성을 생성하고, "30분 만에 문서 생성"이 즉각적 해결책을 제시합니다.

#### 전환 포인트 2: 무료 투명성 고지 문구 생성 후

```
[투명성 고지 문구 생성 완료]
        ↓
"투명성 고지 문구가 생성되었습니다.
 그런데 이것만으로는 부족합니다.

 고영향 AI라면 추가로 필요한 문서:
 - 영향평가서 (Pro에서 자동 생성)
 - 위험관리계획서 (Pro에서 자동 생성)

 [Pro 플랜에서 모든 문서 완성하기]"
```

#### 전환 포인트 3: 문서 미리보기 (Pro-gating)

```
[영향평가서 미리보기]
"이 영향평가서의 처음 1페이지를 확인하세요:
 [미리보기 내용...]

 전체 문서(15페이지)를 확인하고 다운로드하려면
 Pro 플랜이 필요합니다.
 [Pro 플랜으로 업그레이드]"
```

### 2.3 사용량 기반 업그레이드 트리거

| 트리거 | 무료 한도 | 업그레이드 메시지 |
|--------|----------|-----------------|
| 자가진단 횟수 | 1회 | "추가 AI 시스템의 진단이 필요하시면 Pro 플랜에서 무제한 진행 가능합니다" |
| AI 시스템 등록 | 1개 | "Pro 플랜에서 최대 3개 AI 시스템을 관리하세요" |
| 문서 생성 | 투명성 고지문만 | "영향평가서, 위험관리계획서는 Pro 플랜에서 생성 가능합니다" |
| 문서 다운로드 | PDF 미리보기만 | "전체 문서 다운로드는 Pro 플랜에서 가능합니다" |
| 대시보드 | 기본 현황만 | "상세 대시보드 + 타임라인 + 알림은 Pro 플랜에서 제공됩니다" |

### 2.4 In-App 성장 루프

#### 루프 1: 진단 -> 공유 -> 유입 루프

```
사용자가 자가진단 완료
    ↓
결과 카드 자동 생성 + "공유하기" CTA
    ↓
LinkedIn/X/카카오톡에 공유
    ↓
공유 글을 본 동료가 AILEX 방문
    ↓
신규 사용자 자가진단 실행
    ↓ (반복)
```

구현: 결과 페이지에 소셜 공유 버튼 + OG 이미지 자동 생성

#### 루프 2: 초대 -> 팀 확장 루프 (Enterprise)

```
법무팀장이 AILEX Enterprise 사용
    ↓
"팀원 초대하기" 기능으로 CTO, 개발팀장 초대
    ↓
초대받은 팀원이 AILEX 접속 + 활동
    ↓
팀원이 자신의 회사 동료/지인에게 소개
    ↓ (반복)
```

구현: Enterprise 플랜에 "팀원 초대" 기능 + 초대 이메일 자동 발송

#### 루프 3: 컴플라이언스 갱신 루프 (리텐션)

```
사용자가 Pro로 컴플라이언스 완료
    ↓
6개월 후 "컴플라이언스 갱신 리마인더" 알림
  "AI 시스템이 업데이트되었다면 재평가가 필요합니다"
    ↓
재평가 실행 -> 문서 업데이트
    ↓
갱신된 배지 발급
    ↓ (6개월마다 반복)
```

구현: 진단 완료일 기준 6개월 후 이메일 + 인앱 알림

### 2.5 PLG + 세일즈 하이브리드 모델

AILEX는 순수 PLG만으로는 Enterprise 고객을 전환하기 어렵습니다 (규제/컴플라이언스 영역 특성상 의사결정자 설득 필요). 따라서 **PLG + Sales Assist** 하이브리드 모델을 적용합니다.

```
[PLG Zone - 자동화]
  무료 자가진단 -> Aha Moment -> Pro 전환 (셀프서비스)
  대상: AI 스타트업 CTO, 1인 창업자
  목표: Pro 플랜 자동 전환율 5%+

[Sales Assist Zone - 수동]
  무료 자가진단 후 "Enterprise 문의" 클릭 시 -> 세일즈 컨택
  또는 AI 시스템 3개 이상 등록 시 -> 자동 세일즈 알림
  대상: 중견기업 법무팀
  목표: Enterprise 월 2~3건 클로징
```

### 2.6 PLG 핵심 지표

| 지표 | 정의 | 30일 목표 | 90일 목표 |
|------|------|----------|----------|
| PQL (Product Qualified Lead) | 자가진단 완료 + "고영향 AI 해당" 판정 | 100명 | 420명 |
| Time-to-Value | 가입 -> 자가진단 완료까지 시간 | 10분 이내 | 5분 이내 |
| 무료 -> Pro 전환율 | Pro 전환 / 전체 가입자 | 3% | 5% |
| PQL -> Pro 전환율 | Pro 전환 / PQL | 10% | 17% |
| 기능 게이팅 클릭률 | Pro 기능 CTA 클릭 / 무료 사용자 | 15% | 25% |
| 셀프서비스 전환 비율 | 세일즈 없이 Pro 전환 / 전체 Pro 전환 | 50% | 70% |

## Part 3: 커뮤니티 + PLG 통합 플라이휠

```
                    커뮤니티
                   /        \
         교육 콘텐츠          피드백
        (AI 기본법 해설)    (기능 요청/버그)
              |                   |
              v                   v
         인지 + 신뢰        제품 개선
              |                   |
              v                   v
    [무료 자가진단] -----> [Pro 전환]
              |                   |
              v                   v
         결과 공유          배지 + 후기
              |                   |
              v                   v
         신규 유입 --------> 커뮤니티 합류
              |
              v
          (플라이휠 반복)
```

**핵심 원리**: 커뮤니티에서 교육 콘텐츠를 통해 신뢰를 쌓고, PLG(무료 자가진단)로 제품 체험을 유도하며, 만족한 사용자가 커뮤니티에서 후기/추천을 통해 신규 사용자를 유입시키는 선순환 구조를 구축합니다.
