# AILEX 퍼널 최적화 실험 + ICE 우선순위

> 작성일: 2026-02-14
> 프레임워크: AARRR 퍼널 (Awareness -> Acquisition -> Activation -> Retention -> Revenue -> Referral)
> 원칙: 가설 기반, 측정 가능, 1인 실행 가능 (주 5시간 이내)

---

## AARRR 퍼널 기본 구조 (AILEX 맞춤)

```
Awareness (인지)
  "AI 기본법 대응이 필요하다는 것을 인식"
  지표: 웹사이트 방문자 수, 브랜드 검색량
        ↓ 목표 전환율: 8%
Acquisition (획득)
  "AILEX에 가입"
  지표: 가입자 수, 가입 전환율
        ↓ 목표 전환율: 70%
Activation (활성화)
  "무료 자가진단 완료 = Aha Moment"
  지표: 자가진단 실행률, 완료율
        ↓ 목표 전환율: 50% (고영향 판정)
Retention (유지)
  "반복 방문 + 추가 진단/문서 업데이트"
  지표: 주간 활성 사용자(WAU), 재방문율
        ↓ 목표 전환율: 17% (고영향 판정자 중)
Revenue (수익)
  "Pro/Enterprise 유료 전환"
  지표: 유료 전환율, MRR, ARPU
        ↓ 목표: 10%가 추천
Referral (추천)
  "다른 기업에게 AILEX 추천/공유"
  지표: 추천 수, K-factor, NPS
```

## 퍼널 실험 10개

### 실험 1: Awareness -> Acquisition
**실험명**: AI 기본법 과태료 계산기 리드 마그넷

**가설**: "AI 기본법 과태료 자동 계산기"를 무료 도구로 제공하면, 블로그/SEO를 통한 웹사이트 방문자의 가입 전환율이 현재 예상치 8%에서 12%로 50% 개선될 것이다.

**변형**:
- A (현재): 블로그 글 하단에 "무료 자가진단" CTA
- B (실험): 블로그 글 내에 "과태료 계산기" 인터랙티브 도구 삽입 -> 결과 확인 시 이메일 입력 요구

**성공 기준**: 가입 전환율 12% 이상 (현재 예상 8%)
**기간**: 2주
**측정**: Google Analytics 이벤트 + Supabase 가입 수
**필요 리소스**: 과태료 계산 로직 구현 (1일), 블로그 임베드 (반일)

---

### 실험 2: Awareness -> Acquisition
**실험명**: LinkedIn 카루셀 콘텐츠 A/B 테스트

**가설**: AI 기본법 해설을 카루셀(슬라이드) 형태로 LinkedIn에 게시하면, 텍스트 전용 포스트 대비 클릭률이 2배 이상 높아지고, 주 10건 이상의 인바운드 문의가 발생할 것이다.

**변형**:
- A: 텍스트 전용 LinkedIn 포스트 (주 3회)
- B: 카루셀(5~10장 슬라이드) LinkedIn 포스트 (주 3회)

**성공 기준**: 카루셀 포스트의 평균 impression이 텍스트 대비 2배, CTA 클릭률 3% 이상
**기간**: 4주 (각 2주씩 교대)
**측정**: LinkedIn Analytics (impression, 클릭, 프로필 조회)
**필요 리소스**: Canva로 카루셀 제작 (1시간/편)

---

### 실험 3: Acquisition -> Activation
**실험명**: 온보딩 프로그레스 바 + 즉시 자가진단 유도

**가설**: 가입 후 첫 화면에서 프로그레스 바("1/3 완료: 자가진단 진행하기")를 표시하면, 가입 후 24시간 내 자가진단 실행률이 50%에서 70%로 개선될 것이다.

**변형**:
- A (현재): 가입 후 대시보드 표시 -> 사용자가 자가진단 메뉴를 직접 찾아야 함
- B (실험): 가입 후 프로그레스 바 + "첫 번째 단계: 무료 자가진단" 버튼 강조

**성공 기준**: 가입 후 24시간 내 자가진단 실행률 70% 이상
**기간**: 2주
**측정**: PostHog 퍼널 분석 (가입 -> 자가진단 시작 -> 자가진단 완료)
**필요 리소스**: 온보딩 UI 수정 (반일)

---

### 실험 4: Acquisition -> Activation
**실험명**: Time-to-Value 최소화 -- "60초 미리보기"

**가설**: 가입 없이도 AI 기본법 자가진단을 "60초 미리보기"로 체험할 수 있게 하면, 가입 전환율이 8%에서 15%로 개선될 것이다.

**변형**:
- A (현재): 자가진단은 가입 후에만 가능
- B (실험): 랜딩 페이지에서 3개 질문만 답하면 "간이 판정 결과" 표시 -> "상세 결과를 보시려면 가입하세요" CTA

**성공 기준**: 랜딩 페이지 방문 -> 가입 전환율 15% 이상
**기간**: 2주
**측정**: GA4 전환 추적 + PostHog
**필요 리소스**: 간이 판정 로직 (반일), 랜딩 페이지 수정 (반일)

---

### 실험 5: Activation -> Retention
**실험명**: AI 기본법 뉴스레터 + 리인게이지먼트 이메일

**가설**: 자가진단 완료 후 비활성 사용자에게 "AI 기본법 주간 업데이트" 뉴스레터를 발송하면, 30일 재방문율이 20%에서 40%로 개선될 것이다.

**변형**:
- A (현재): 자가진단 완료 후 별도 리인게이지먼트 없음
- B (실험): 주 1회 "AI 기본법 이번 주 변경사항/뉴스 + 당신의 진단 결과 리마인더" 이메일 발송

**성공 기준**: 이메일 발송 후 7일 내 재방문율 40% 이상
**기간**: 4주
**측정**: Resend 오픈율/클릭률 + 재방문 세션 수
**필요 리소스**: 뉴스레터 템플릿 설정 (1시간), 주간 작성 (30분/주)

---

### 실험 6: Activation -> Retention
**실험명**: 습관 형성 -- "컴플라이언스 건강 점수"

**가설**: 대시보드에 "컴플라이언스 건강 점수"(0~100점)를 표시하고, 점수가 낮을 때 인앱 알림을 보내면, 주간 활성 사용률(WAU)이 15%에서 30%로 개선될 것이다.

**변형**:
- A (현재): 정적 대시보드 (문서 목록만 표시)
- B (실험): 동적 건강 점수 (미완료 의무 = 감점) + "점수를 올리려면 [액션]을 완료하세요" CTA

**성공 기준**: WAU (주 1회 이상 접속) 30% 이상
**기간**: 4주
**측정**: PostHog 리텐션 코호트 분석
**필요 리소스**: 건강 점수 로직 + UI 구현 (1~2일)

---

### 실험 7: Retention -> Revenue
**실험명**: 고영향 AI 판정 후 즉시 업셀 모달

**가설**: 자가진단에서 "고영향 AI 해당" 판정을 받은 사용자에게, 결과 화면에서 즉시 "의무 문서를 지금 생성하세요 -> Pro 플랜" 업셀 모달을 표시하면, 고영향 판정자의 Pro 전환율이 10%에서 20%로 개선될 것이다.

**변형**:
- A (현재): 판정 결과 페이지에 Pro 기능 설명 텍스트
- B (실험): 판정 결과 직후 "지금 바로 영향평가서를 AI로 생성하세요" 모달 + "파운더 플랜 50% 할인" 긴급 CTA

**성공 기준**: 고영향 판정자 중 Pro 전환율 20% 이상
**기간**: 2주
**측정**: PostHog 퍼널 (판정 결과 -> 모달 표시 -> 결제 페이지 -> 결제 완료)
**필요 리소스**: 업셀 모달 UI 구현 (반일)

---

### 실험 8: Retention -> Revenue
**실험명**: 가격 프레이밍 -- "과태료 대비 ROI 표시"

**가설**: 가격 페이지에서 "월 30만원" 대신 "과태료 3,000만원의 1% / 법무법인의 1/67 가격 / 하루 1만원"으로 프레이밍하면, 가격 페이지 -> 결제 전환율이 15%에서 25%로 개선될 것이다.

**변형**:
- A (현재): "Pro 플랜: 월 30만원"
- B (실험): "Pro 플랜: 하루 1만원 (과태료 3,000만원의 1%, 법무법인 비용의 1/67)"

**성공 기준**: 가격 페이지 -> 결제 완료 전환율 25% 이상
**기간**: 2주
**측정**: GA4 + PostHog (가격 페이지 방문 -> 결제 완료)
**필요 리소스**: 가격 페이지 카피 수정 (1시간)

---

### 실험 9: Revenue -> Referral
**실험명**: NPS 기반 추천 요청 타이밍

**가설**: NPS 설문에서 9~10점(Promoter)을 준 사용자에게 즉시 "동료에게 추천하기" CTA를 표시하면, 추천 프로그램 참여율이 10%에서 30%로 개선될 것이다.

**변형**:
- A (현재): 추천 프로그램은 대시보드 메뉴에 위치 (능동적으로 찾아야 함)
- B (실험): NPS 9~10점 응답 직후 "감사합니다! 동료에게도 추천해주시겠어요?" 팝업 + 추천 링크 + 보상 안내

**성공 기준**: NPS Promoter의 추천 참여율 30% 이상
**기간**: 4주 (NPS 수집 + 추천 전환 추적)
**측정**: NPS 응답 -> 추천 링크 생성 -> 피추천인 가입 추적
**필요 리소스**: NPS 설문 인앱 구현 (반일) + 추천 팝업 (반일)

---

### 실험 10: Revenue -> Referral
**실험명**: 유료 사용자 후기/사례 자동 수집

**가설**: Pro 사용자가 첫 의무 문서 생성을 완료한 직후 "1분 후기 작성" 요청을 하면, 후기 수집률이 5%에서 20%로 개선되고, 수집된 후기가 랜딩 페이지에 표시되어 가입 전환율을 10% 향상시킬 것이다.

**변형**:
- A (현재): 후기 수집 없음
- B (실험): 문서 생성 완료 직후 "방금 경험은 어떠셨나요? 한 줄 후기를 남겨주세요" 인앱 모달 + "LinkedIn에도 공유하기" 버튼

**성공 기준**: 후기 수집률 20% 이상, 후기 게시 후 가입 전환율 10% 향상
**기간**: 4주
**측정**: 후기 제출 수 / 문서 생성 수, 후기 게시 전후 가입 전환율 비교
**필요 리소스**: 후기 수집 모달 (반일), 랜딩 페이지 후기 섹션 (반일)
