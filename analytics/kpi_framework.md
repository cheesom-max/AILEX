# AILEX KPI 프레임워크

> 작성일: 2026-02-13
> 기반: mvp_spec.md, gtm_strategy.md
> 제품: AILEX - AI 기본법 컴플라이언스 자동화 B2B SaaS
> 핵심 퍼널: 방문 -> 가입 -> 무료 판정 -> 고영향 AI 해당 -> Pro 전환

---

## 1. North Star Metric

**"주간 의무 문서 생성 완료 건수" (Weekly Compliance Documents Generated)**

### 정의
Pro/Enterprise 유료 사용자가 1주일간 실제로 생성 완료한 의무 문서(영향평가서, 투명성 보고서, 위험관리계획서) 총 건수.

### 선정 근거
- AILEX의 핵심 가치는 "컴플라이언스 문서를 자동으로 만들어주는 것"이다
- 문서가 생성된다는 것은 (1) 유료 전환이 되었고 (2) 제품의 핵심 기능을 실제 사용하고 있으며 (3) 고객이 비즈니스 가치를 얻고 있다는 증거다
- 이 지표가 올라가면: 리텐션 상승, MRR 증가, NPS 개선이 자연스럽게 따라온다
- 이 지표가 내려가면: 제품-시장 적합성(PMF) 문제, 기능 품질 문제, 또는 온보딩 실패를 즉시 의심해야 한다

### 계산
```
Weekly Documents Generated = SUM(completed_documents) WHERE created_at BETWEEN [이번 주 월요일] AND [이번 주 일요일]
```

### 목표
| 시점 | 목표 | 근거 |
|------|------|------|
| 30일 | 주 15건 | 유료 10명 x 주 1.5건 |
| 60일 | 주 60건 | 유료 30명 x 주 2건 |
| 90일 | 주 150건 | 유료 75명 x 주 2건 |

## 2. 핵심 지표 (AARRR 프레임워크)

### 2.1 Acquisition: 주간 신규 가입자 수 (Weekly New Signups)

**정의**: 해당 주에 이메일 또는 Google OAuth로 가입을 완료한 신규 사용자 수 (중복 제외).

**계산**:
```
Weekly New Signups = COUNT(DISTINCT user_id) WHERE created_at BETWEEN [주 시작] AND [주 끝]
```

**채널별 세분화**:
- Organic Search (SEO/블로그)
- LinkedIn (콘텐츠 + DM)
- Direct (직접 URL 입력/북마크)
- Referral (공유 링크/배지)
- Email (콜드 이메일/뉴스레터)
- Community (디스콰이엇/GeekNews)
- Seminar (오프라인 행사 QR)

**목표값**:
| 시점 | 목표 | 주간 환산 | 의사결정 기준 |
|------|------|-----------|--------------|
| 30일 | 300명 누적 | 주 75명 | 주 50명 미만이면 채널 전략 재검토 |
| 60일 | 700명 누적 | 주 100명 | 특정 채널이 70% 이상이면 채널 다각화 필요 |
| 90일 | 1,200명 누적 | 주 125명 | 가입 전환율 8% 미만이면 랜딩페이지 A/B 테스트 |

**측정 방법**: Supabase DB `profiles` 테이블 + PostHog `$signup` 이벤트 + UTM 파라미터 기반 채널 구분

**의사결정 연결**:
- 주간 가입 50명 미만 -> SEO 콘텐츠 발행 빈도 2배 증가 + 콜드 이메일 볼륨 증가
- 특정 채널 CAC가 평균의 3배 이상 -> 해당 채널 중단 또는 메시지 변경
- 가입 전환율(방문 대비) 5% 미만 -> 랜딩페이지 히어로 섹션/CTA A/B 테스트

---

### 2.2 Activation: 무료 판정 완료율 (Free Assessment Completion Rate)

**정의**: 가입 후 7일 이내에 무료 자가진단(고영향 AI 판정)을 1회 이상 완료한 사용자 비율.

**계산**:
```
Activation Rate = (가입 후 7일 내 판정 완료 사용자 수) / (해당 기간 가입자 수) x 100
```

**목표값**:
| 시점 | 목표 | 의사결정 기준 |
|------|------|--------------|
| 30일 | 60% | 50% 미만이면 온보딩 플로우 개선 (가입 직후 판정 유도 모달 추가) |
| 60일 | 65% | 60% 미만이면 온보딩 이메일 시퀀스 강화 (Day 1 판정 안내 이메일) |
| 90일 | 70% | GTM 전략의 목표 70%와 일치 |

**측정 방법**: PostHog 커스텀 이벤트 `assessment_completed` + 코호트별 퍼널 분석

**세부 퍼널 단계**:
```
가입 완료 (100%)
  -> 대시보드 방문 (목표: 90%)
  -> "자가진단 시작" 클릭 (목표: 80%)
  -> AI 시스템 정보 입력 (목표: 75%)
  -> 판정 결과 확인 (목표: 70%)
```

**의사결정 연결**:
- "자가진단 시작" 클릭률 70% 미만 -> CTA 위치/문구 변경, 대시보드에서 자가진단 카드 강조
- 입력 중 이탈률 20% 이상 -> 입력 폼 단순화, 예시 데이터 프리필 옵션 제공
- 판정 결과까지 도달했으나 활성화 판정이 안 된 경우 -> 판정 로딩 시간 개선, 결과 화면 UX 개선

---

### 2.3 Retention: Week 4 리텐션율 (W4 Retention)

**정의**: 가입 후 4주차에 AILEX에 재방문하여 1회 이상 의미 있는 액션(문서 확인/편집/생성, 대시보드 조회, 새 AI 시스템 등록)을 수행한 사용자 비율.

**계산**:
```
W4 Retention = (가입 4주차에 의미 있는 액션을 수행한 사용자 수) / (4주 전 가입자 수) x 100
```

**코호트별 추적**:
- Free 사용자 W4 리텐션 목표: 25%
- Pro 사용자 W4 리텐션 목표: 85%
- "고영향 판정" 받은 Free 사용자 W4 리텐션 목표: 40%

**목표값**:
| 시점 | 전체 W4 리텐션 | Pro W4 리텐션 | 의사결정 기준 |
|------|---------------|--------------|--------------|
| 60일 | 30% | 80% | Pro 리텐션 70% 미만이면 문서 품질 점검 + 1:1 고객 인터뷰 |
| 90일 | 35% | 85% | Free 리텐션 20% 미만이면 재방문 유인 부족 (이메일 넛지 강화) |

**측정 방법**: PostHog 코호트 리텐션 차트 + Supabase 로그인/액션 로그

**의사결정 연결**:
- Free W4 리텐션 20% 미만 -> 무료 사용자에게 월간 컴플라이언스 뉴스레터/업데이트 발송
- Pro W4 리텐션 70% 미만 -> 이탈 위험 고객 개별 연락, 문서 생성 품질 문제 확인
- "고영향 판정" Free 사용자 리텐션이 낮으면 -> Free-to-Pro 전환 시점/메시지 최적화 필요

---

### 2.4 Revenue: MRR (Monthly Recurring Revenue)

**정의**: 해당 월의 반복 발생 매출 합계.

**계산**:
```
MRR = (Pro 사용자 수 x 30만원) + (Enterprise 사용자 수 x 100만원)
     + (파운더 플랜 사용자 수 x 15만원)
```

**세분화**:
- New MRR: 신규 유료 전환에서 발생한 MRR
- Expansion MRR: Pro -> Enterprise 업그레이드에서 발생한 추가 MRR
- Contraction MRR: Enterprise -> Pro 다운그레이드로 감소한 MRR
- Churned MRR: 유료 해지로 사라진 MRR
- Net New MRR = New + Expansion - Contraction - Churned

**목표값**:
| 시점 | MRR 목표 | Pro 수 | Enterprise 수 | 의사결정 기준 |
|------|----------|--------|---------------|--------------|
| 30일 | 150만원 | 10명 | 0명 | Net New MRR이 음수면 즉시 이탈 원인 분석 |
| 60일 | 450만원 | 30명 | 2명 | 파운더 플랜 전환율이 전체 3% 미만이면 프로모션 강화 |
| 90일 | 1,500만원+ | 70명 | 5명 | Churned MRR이 New MRR의 30% 이상이면 리텐션 위기 |

**측정 방법**: Supabase DB `profiles.plan` + 결제 시스템(Stripe) + 수동 스프레드시트 (Enterprise)

**의사결정 연결**:
- Net New MRR 3개월 연속 감소 -> 가격 전략 재검토, 고객 인터뷰 집중 실시
- Enterprise 전환 0건 (60일차 이후) -> Enterprise 전용 기능/가치 제안 강화, 직접 영업 강화
- Churned MRR/Total MRR 비율 10% 초과 -> 이탈 방지 캠페인 즉시 가동

---

### 2.5 Referral: 공유 전환율 (Share-to-Signup Rate)

**정의**: 판정 결과 공유(LinkedIn/X/카카오톡/배지)를 통해 유입된 신규 가입자 비율.

**계산**:
```
Share-to-Signup Rate = (공유 링크를 통해 가입한 사용자 수) / (공유 버튼 클릭 수) x 100
```

**바이럴 계수 (K-factor)**:
```
K-factor = (사용자당 평균 공유 횟수) x (공유당 가입 전환율)
목표: K = 0.3 이상 (GTM 전략 기준)
```

**목표값**:
| 시점 | 공유 클릭률 | 공유 -> 가입 전환 | K-factor | 의사결정 기준 |
|------|-----------|-----------------|----------|--------------|
| 30일 | 5% | 15% | 0.08 | 기준선 확보 단계 |
| 60일 | 8% | 20% | 0.16 | 공유 인센티브(추가 진단 1회) 테스트 |
| 90일 | 10% | 25% | 0.30 | K < 0.15면 공유 UX/인센티브 전면 재설계 |

**측정 방법**: PostHog 커스텀 이벤트 `result_shared` + UTM `source=share` + `ref` 파라미터

**의사결정 연결**:
- 공유 버튼 클릭률 3% 미만 -> 공유 CTA 위치/디자인 변경, 배지 이미지 매력도 개선
- 공유 -> 가입 전환 10% 미만 -> 공유 랜딩페이지 최적화, 공유 메시지 자동 생성 개선
- "컴플라이언스 인증 배지" 삽입 기업 수도 보조 지표로 추적

## 3. 건강 지표 (Health Metrics)

### 3.1 Monthly Churn Rate (월간 이탈률)

**정의**: 해당 월 초 유료 사용자 중 월말까지 해지한 사용자 비율.

**계산**:
```
Monthly Churn Rate = (월 중 해지한 유료 사용자 수) / (월초 유료 사용자 수) x 100
```

**벤치마크**: B2B SaaS 평균 월 3.5% (Recurly 2025). SMB 대상은 3-7%.

**목표**:
| 시점 | 목표 | 경고 기준 |
|------|------|----------|
| 60일 | 10% 이하 | 15% 초과 시 긴급 대응 |
| 90일 | 8% 이하 | 12% 초과 시 긴급 대응 |
| 6개월 | 5% 이하 | 8% 초과 시 가격/제품 전면 재검토 |

**의사결정 연결**:
- Churn 10% 초과 -> 이탈 사용자 전원 이메일 설문 + 상위 3명 1:1 인터뷰
- Involuntary Churn(결제 실패) 비중 높으면 -> 결제 실패 알림 + 재시도 로직 강화
- 특정 코호트(파운더 플랜 만료 후)의 이탈이 높으면 -> 갱신 할인 프로모션

---

### 3.2 CAC (Customer Acquisition Cost)

**정의**: 1명의 유료 고객을 획득하기 위해 투입한 총 비용.

**계산**:
```
CAC = (마케팅 비용 + 영업 시간 비용) / (해당 기간 신규 유료 고객 수)

마케팅 비용: 도구 비용 + 세미나 비용 + 콘텐츠 제작 시간 비용
영업 시간 비용: 콜드 이메일/DM 시간 x 시급 환산
```

**채널별 CAC 추적**:
| 채널 | 예상 CAC | 측정 방법 |
|------|---------|----------|
| SEO/블로그 | 5만원 이하 | 콘텐츠 작성 시간 환산 / SEO 유입 전환 수 |
| LinkedIn | 10만원 | 포스팅 시간 + DM 시간 / LinkedIn 유입 전환 수 |
| 세미나 | 15만원 | 행사 비용 + 시간 / 행사 유입 전환 수 |
| 콜드 이메일 | 8만원 | 이메일 작성/발송 시간 / 이메일 유입 전환 수 |
| Referral | 2만원 이하 | 공유 인센티브 비용 / 공유 유입 전환 수 |

**목표**: CAC 20만원 이하 (Pro 30만원 기준, 1개월 이내 회수)

---

### 3.3 LTV (Customer Lifetime Value)

**정의**: 1명의 유료 고객이 전체 이용 기간 동안 가져오는 총 매출.

**계산**:
```
LTV = ARPU x (1 / Monthly Churn Rate)

ARPU (Average Revenue Per User):
  - Pro 전용: 30만원
  - 파운더 플랜: 15만원
  - 혼합: (Pro 수 x 30만 + Enterprise 수 x 100만 + 파운더 수 x 15만) / 전체 유료 수
```

**목표**:
| 시점 | ARPU | 예상 Churn | LTV | LTV/CAC |
|------|------|-----------|-----|---------|
| 90일 | 25만원 | 8% | 312만원 | 15x 이상 |
| 6개월 | 28만원 | 5% | 560만원 | 28x 이상 |

---

### 3.4 LTV/CAC Ratio

**정의**: 고객 생애 가치 대비 고객 획득 비용 비율.

**계산**:
```
LTV/CAC Ratio = LTV / CAC
```

**벤치마크**: 건강한 SaaS는 3x 이상, 우수한 SaaS는 5x 이상.

**목표**: 초기 5x 이상 유지 (CAC 20만원 이하, LTV 100만원 이상)

**경고 기준**:
- LTV/CAC 3x 미만 -> 마케팅 채널 효율 점검 + 가격 인상 검토
- LTV/CAC 10x 이상 -> 마케팅 투자가 부족한 것, 채널 확대 필요

---

### 3.5 Burn Rate & Runway

**정의**: 월간 순 현금 소진량과 현재 자금으로 운영 가능한 개월 수.

**계산**:
```
Monthly Burn Rate = 월 지출 - 월 수입
Runway (개월) = 보유 자금 / Monthly Burn Rate
```

**초기 월 지출 추정**:
| 항목 | 비용 |
|------|------|
| AI API (GPT-4o) | ~10만원 |
| Vercel Pro | ~3만원 |
| Supabase Pro | ~4만원 |
| 도메인/기타 | ~2만원 |
| 마케팅 비용 | ~25만원 |
| **합계** | **~44만원/월** |

**경고 기준**: Runway 3개월 미만이면 비용 절감 또는 자금 조달 필요

---

## 4. 전체 퍼널 전환율 목표

GTM 전략의 90일 퍼널 목표를 단계별로 추적한다.

```
웹사이트 방문 (15,000)
    | 가입 전환율: 8% -------- [경고: 5% 미만]
    v
가입자 (1,200)
    | 무료 판정 실행율: 70% --- [경고: 50% 미만]
    v
판정 실행 (840)
    | "고영향 AI 해당" 비율: 50% (시장 특성, 통제 불가)
    v
고영향 판정 (420)
    | Pro 전환율: 17% --------- [경고: 10% 미만]
    v
유료 전환 (70~75)
    + Enterprise 직접 영업 (5)
    = 총 유료 고객 75~80명 (90일 차)

    | Pro W4 리텐션: 85% ------ [경고: 70% 미만]
    v
MRR 1,500만원+ 달성
```

### 퍼널 단계별 전환 목표 및 진단 표

| 퍼널 단계 | 전환 지표 | 30일 목표 | 60일 목표 | 90일 목표 | 경고 기준 | 경고 시 액션 |
|-----------|----------|----------|----------|----------|----------|-------------|
| 방문 -> 가입 | 가입 전환율 | 6% | 7% | 8% | 5% 미만 | 랜딩페이지 CTA/히어로 A/B 테스트 |
| 가입 -> 판정 실행 | 활성화율 | 60% | 65% | 70% | 50% 미만 | 온보딩 플로우 개선, 이메일 넛지 추가 |
| 판정 -> 고영향 판정 | 고영향 비율 | 50% | 50% | 50% | - | 시장 특성, 통제 불가 (타겟팅 정밀도로 간접 영향) |
| 고영향 판정 -> Pro | Pro 전환율 | 12% | 15% | 17% | 10% 미만 | 전환 CTA/메시지 강화, 파운더 플랜 프로모션 강조 |
| Pro -> W4 리텐션 | W4 리텐션 | - | 80% | 85% | 70% 미만 | 문서 품질 개선, 1:1 고객 인터뷰, 온보딩 이메일 |

## 5. 경고 기준 (Alert Thresholds) 요약

즉시 대응이 필요한 상황을 한눈에 파악할 수 있도록 정리한다.

| 경고 등급 | 조건 | 대응 시간 | 대응 행동 |
|----------|------|----------|----------|
| **긴급 (RED)** | Monthly Churn > 15% | 24시간 | 이탈 고객 전원 연락, 제품 결함 점검 |
| **긴급 (RED)** | Net New MRR 음수 (2주 연속) | 48시간 | 가격/제품/영업 전략 긴급 회의 |
| **긴급 (RED)** | Pro W4 리텐션 < 60% | 48시간 | 유료 고객 전원 인터뷰, 핵심 기능 점검 |
| **경고 (YELLOW)** | 주간 신규 가입 < 50명 | 1주 이내 | 마케팅 채널 전략 재검토 |
| **경고 (YELLOW)** | 활성화율 < 50% | 1주 이내 | 온보딩 UX 개선 |
| **경고 (YELLOW)** | Pro 전환율 < 10% | 1주 이내 | 전환 CTA/프로모션 A/B 테스트 |
| **경고 (YELLOW)** | LTV/CAC < 3x | 2주 이내 | CAC 높은 채널 중단 또는 가격 인상 검토 |
| **주의 (GREEN)** | NPS < 30 | 월 단위 | 고객 피드백 수집 강화, 불만 사항 우선 개선 |
| **주의 (GREEN)** | 공유 K-factor < 0.1 | 월 단위 | 바이럴 기능/인센티브 실험 |
