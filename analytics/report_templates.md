## 4. 의사결정 로그 (Decision Log)

### 4.1 목적

모든 데이터 기반 의사결정을 기록하여 "왜 그랬는지"를 추적하고, 결과를 학습한다.
1인 운영에서 가장 쉽게 놓치는 것이 "과거의 나는 왜 이 결정을 했는가"이다.

### 4.2 템플릿

```markdown
# AILEX 의사결정 로그

## [YYYY-MM-DD] [결정 제목]

### 상황
[어떤 데이터/상황이 의사결정을 촉발했는가]

### 선택지
1. [옵션 A]: [장단점]
2. [옵션 B]: [장단점]
3. [옵션 C]: [장단점] (있다면)

### 결정
[옵션 X]를 선택한다.

### 근거 데이터
- [구체적 수치/차트/출처]
- [구체적 수치/차트/출처]

### 예상 결과
[이 결정으로 인해 X 지표가 Y%만큼 개선될 것으로 예상]

### 실제 결과 (추후 기입)
- 측정 시점: [날짜]
- 결과: [수치]
- 예상 대비: [일치/초과/미달]

### 학습
[이 의사결정에서 배운 것]

---
```

### 4.3 예시

```markdown
## 2026-03-15 콜드 이메일 채널 중단, LinkedIn DM으로 전환

### 상황
콜드 이메일 4주 운영 결과 응답률 3% (목표 10%의 30%). 반면 LinkedIn DM
응답률 18%, Pro 전환율도 이메일(2%) 대비 DM(8%)이 4배 높음.
시간 투입은 두 채널 모두 주 3시간으로 동일.

### 선택지
1. 콜드 이메일 계속 (제목/내용 A/B 테스트로 개선 시도)
2. 콜드 이메일 중단, LinkedIn DM 시간 2배로 확대
3. 두 채널 모두 유지하되 이메일 볼륨 절반으로 축소

### 결정
[옵션 2] 콜드 이메일 중단, LinkedIn DM 주 3시간 -> 주 6시간으로 확대

### 근거 데이터
- 콜드 이메일 CAC: 45만원 (Pro 1명 전환에 이메일 200통, 시간 20시간)
- LinkedIn DM CAC: 11만원 (Pro 1명 전환에 DM 30건, 시간 6시간)
- CAC 차이 4배, 같은 시간 투입 시 DM이 4배 효율적

### 예상 결과
주간 Pro 전환 2명 -> 4명으로 증가 (DM 볼륨 2배 = 전환 2배 가정)

### 실제 결과 (2026-04-15 기입)
- 주간 Pro 전환: 평균 3.5명 (예상 4명의 88%)
- DM 응답률이 볼륨 증가에 따라 15%로 소폭 하락 (포화 효과)
- 그래도 이메일 대비 2배 이상 효율적

### 학습
DM 볼륨 확대 시 응답률 소폭 하락은 자연스러움.
다만 일 15건 이상이면 스팸 느낌을 줄 수 있어 일 10건 유지가 적정.
```

### 6.1 Phase 1 (수동, 런칭 ~ 30일)

모든 리포트를 수동으로 작성한다. 도구 간 이동이 필요하지만, 지표와 데이터에 대한 깊은 이해를 얻는 기간.

**주간 루틴**:
```
월요일 09:00: 주간 리포트 작성 (PostHog -> Stripe -> MD 파일)
```

**월간 루틴**:
```
매월 1일: 월간 리포트 작성 (PostHog + GA4 + Stripe + Supabase SQL -> MD 파일)
```

### 6.2 Phase 2 (반자동, 30일 ~ 90일)

PostHog의 Subscription 기능으로 주간 대시보드 스냅샷을 이메일로 자동 발송.

```
PostHog -> Dashboard -> "AILEX Weekly Growth"
-> Subscribe -> Weekly, Monday 09:00 KST
-> 이메일로 대시보드 스냅샷 수신
-> 스냅샷 기반으로 MD 리포트 빠르게 작성 (10분으로 단축)
```

### 6.3 Phase 3 (자동, 90일 이후)

Supabase Edge Function으로 주간 리포트 자동 생성을 고려한다.


## 8. 리포트 작성 시 주의사항

### 하지 말아야 할 것

1. **허영 지표 보고**: "이번 주 페이지뷰 50,000!" -> 의미 없음. 전환과 연결된 지표만 보고.
2. **수치만 나열**: "가입 125명, MRR 1,350만원" -> 그래서 뭘 해야 하는가? 항상 "액션"을 포함.
3. **변명 작성**: "LinkedIn 알고리즘 변경으로 도달이 줄었다" -> 통제 불가한 원인 분석보다 통제 가능한 대안을 제시.
4. **완벽한 리포트 작성에 1시간 이상 소비**: 15분 안에 끝나지 않으면 리포트가 너무 복잡한 것. 간소화.

### 반드시 해야 할 것

1. **전주 대비 변화**: 절대값보다 변화율이 중요. "+25%"가 "125명"보다 의미 있다.
2. **목표 대비 진행률**: 90일 목표에 대해 현재 어디에 있는지 항상 표시.
3. **1개의 핵심 인사이트**: 매주 1개만이라도 "이번 주에 발견한 것"을 기록.
4. **1-2개의 구체적 액션**: "개선해야 한다"가 아닌 "월요일에 X를 한다".
5. **정직한 기록**: 목표 미달도 솔직하게 기록. 미래의 자신이 학습하기 위한 자료.
