## 2. 경영 대시보드 (Daily Executive Dashboard)

**구현 위치**: PostHog 대시보드 "AILEX Daily"
**확인 시간**: 매일 오전 09:00, 5분

### 2.1 위젯 배치도

```
+----------------------------------------------------------+
|                  AILEX 경영 대시보드 (일간)                 |
+----------------------------------------------------------+
|                                                          |
|  [1. North Star]          [2. 오늘의 숫자]                |

|  | 목표: 150 | 달성: XX% |  | 오늘 Pro 전환: X명        |  |
|  | [7일 추이 스파크라인] |  | 활성 Pro 사용자: XX명     |  |

|  +----------------------------------------------------+  |
```

### 2.2 위젯 상세 사양

#### 위젯 1: North Star Metric

| 항목 | 설정 |
|------|------|
| 지표 | 주간 의무 문서 생성 완료 건수 |
| 시각화 | 대형 숫자 (Bold Number) + 7일 스파크라인 |
| 비교 기간 | 전주 대비 변화율 (%, 화살표) |
| PostHog 설정 | Insight -> Trends -> Event: `document_generation_completed`, 기간: Last 7 days |
| 색상 규칙 | 목표 100% 이상: 초록, 70-99%: 노랑, 70% 미만: 빨강 |

#### 위젯 2: 오늘의 숫자

| 항목 | 설정 |
|------|------|
| 지표 (4개) | 오늘 가입, 오늘 판정, 오늘 Pro 전환, 활성 Pro 수 |
| 시각화 | 숫자 카드 4개 (Number Insight) |
| PostHog 설정 | 각각 Insight -> Trends -> 해당 이벤트, 기간: Today |
| 비교 | 어제 동 시간 대비 |

#### 위젯 3: 경고 상태

| 항목 | 설정 |
|------|------|
| 지표 | kpi_framework.md 5. 경고 기준 요약 기반 |
| 시각화 | 텍스트 기반 상태 표시 (RED / YELLOW / GREEN) |
| 구현 | PostHog 대시보드 내 텍스트 위젯 + 수동 업데이트 (자동화는 Phase 2) |
| 대안 | PostHog Alerts 기능으로 이메일/Slack 자동 알림 설정 |

**PostHog Alert 설정 방법**:

```
1. PostHog -> Insights -> 경고 대상 Insight 생성
   - 예: "주간 가입 수" Trends (Last 7 days, signup_completed)
2. Alert 탭 -> Create Alert
   - Condition: "Value is below 50"
   - Notification: Email (본인 이메일)
   - Check interval: Daily
3. 동일하게 각 경고 기준별 Alert 생성
```

#### 위젯 4: 핵심 퍼널 스냅샷

| 항목 | 설정 |
|------|------|
| 지표 | 5단계 전환 퍼널의 각 단계별 전환율 |
| 시각화 | 수평 막대 차트 (Funnel) |
| PostHog 설정 | Insight -> Funnels -> 5개 Step 정의 |
| 기간 | 최근 7일 |
| 전환 기준 | 7일 이내 완료 |

### 4.2 위젯 상세 사양

#### 위젯 1: MRR 현황 및 추이

| 항목 | 설정 |
|------|------|
| 지표 | MRR 절대값 + 전월 대비 변화율 |
| 시각화 | 대형 숫자 + 라인 차트 (최근 6개월) |
| 데이터 소스 | Stripe Dashboard: Revenue -> MRR |
| 보조 데이터 | Supabase SQL MRR 계산 쿼리 (tool_setup_guide.md 참조) |
| 목표선 표시 | 30일: 150만, 60일: 450만, 90일: 1,500만 (점선) |

#### 위젯 2: MRR 구성 분석

| 항목 | 설정 |
|------|------|
| 지표 | New / Expansion / Contraction / Churned MRR |
| 시각화 | 적층 막대 차트 (Stacked Bar, 월별) |
| 데이터 소스 | Supabase `subscription_events` 테이블 집계 |
| 주요 관찰점 | Churned MRR이 New MRR의 30% 이상이면 리텐션 위기 |

**Supabase SQL 쿼리**:

```sql
-- 월별 MRR 구성 분석
SELECT
  date_trunc('month', created_at) as month,
  SUM(mrr_amount) FILTER (WHERE type = 'new') as new_mrr,
  SUM(mrr_amount) FILTER (WHERE type = 'expansion') as expansion_mrr,
  SUM(ABS(mrr_amount)) FILTER (WHERE type = 'contraction') as contraction_mrr,
  SUM(ABS(mrr_amount)) FILTER (WHERE type = 'churned') as churned_mrr,
  SUM(mrr_amount) as net_new_mrr
FROM subscription_events
GROUP BY date_trunc('month', created_at)
ORDER BY month DESC
LIMIT 6;
```

#### 위젯 3: Churn 분석

| 항목 | 설정 |
|------|------|
| 지표 | 월간 이탈률, 이탈 고객 수, Churned MRR, 이탈 사유 분포 |
| 시각화 | 숫자 카드 + 파이 차트 (이탈 사유) |
| 데이터 소스 | Stripe (이탈 수) + Supabase (이탈 설문 결과) |
| 이탈 사유 수집 | 해지 시 "해지 이유" 설문 (PostHog Survey 또는 인앱 모달) |

**이탈 사유 수집 코드 예시**:

```typescript
// 해지 확인 모달에서
const cancelReasons = [
  'no_longer_needed',     // 더 이상 필요하지 않음
  'too_expensive',        // 가격이 부담됨
  'missing_features',     // 필요한 기능 부족
  'poor_quality',         // 문서 품질 불만족
  'competitor',           // 다른 솔루션으로 전환
  'other',               // 기타
]

async function handleCancellation(reason: string, feedback?: string) {
  // PostHog 이벤트
  trackEvent('subscription_cancelled', {
    reason,
    feedback,
    tenure_days: daysSinceSignup,
    plan: currentPlan,
  })

  // Supabase에 저장
  await supabase.from('churn_surveys').insert({
    user_id: userId,
    reason,
    feedback,
    plan: currentPlan,
    tenure_days: daysSinceSignup,
  })
}
```

| 데이터 소스 | Supabase SQL |

```sql
SELECT
  plan,
  COUNT(*) as customer_count,
  ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER(), 1) as percentage
FROM profiles
WHERE plan IN ('pro', 'enterprise', 'founder')
  AND subscription_status = 'active'
GROUP BY plan
ORDER BY customer_count DESC;
```
