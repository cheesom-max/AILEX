## GeekNews Show 게시 가이드

GeekNews Show는 "직접 만든 것"을 공유하는 공간으로, Hacker News의 Show HN과 동일한 철학을 따릅니다.

### 핵심 원칙
- **마케팅/영업 톤 금지**: 사실적이고 직접적인 언어 사용
- **기술적 세부사항 우선**: 어떻게 만들었는지, 왜 이렇게 설계했는지 공유
- **개인 스토리 중심**: 회사 홍보보다 메이커의 개인적 여정
- **겸손한 태도**: "완성되지 않았습니다", "피드백 환영합니다"

## Show GN 포스트 (3가지 버전)

### 버전 1: 기술 중심 (추천)

**제목**: Show GN: AILEX - AI 기본법 고영향 판정을 GPT-4o로 자동화했습니다

**본문**:
안녕하세요.

AI 기본법이 시행되면서 "우리 AI가 고영향에 해당하는지" 판정하는 게 모든 AI 기업의 숙제가 되었습니다. 법 조문 55개를 읽고 11개 영역(고용, 교육, 금융, 의료 등)에 해당하는지 판단해야 하는데, 법률 전문가가 아니면 해석 자체가 어렵습니다.

그래서 GPT-4o로 판정을 자동화하는 도구를 만들었습니다.

**기술적 접근**:
- Next.js 15 + Supabase로 풀스택 구현
- GPT-4o API에 AI 기본법 시행령 전문을 시스템 프롬프트로 주입
- 과학기술정보통신부 가이드라인 30개 예시를 Few-shot Learning
- 3단계 판정 ("확실히 고영향" / "일반" / "모호함") 으로 법적 리스크 최소화
- 테스트 결과 정확도 95%+ 달성

**도전 과제**:
1. 법률 언어의 모호성 처리 (예: "상당한 규모", "중요한 영향" 등 정량 기준 부재)
2. AI 판정의 신뢰성 확보 (면책 고지 + 전문가 검토 권장 필수)
3. 비용 효율화 (판정 1건당 GPT-4o API 비용 $0.30, Prompt Caching으로 90% 절감)

**현재 상태**:
- 베타 운영 중, 10개 스타트업 테스트 완료
- 고영향 판정 외에 AI 영향평가서 자동 생성 기능도 구현
- 법무법인 컨설팅 건당 2,000만원 vs 월 30만원 구독 모델

**피드백 환영**:
- 법률 AI의 정확도 검증 방법에 대한 의견
- 프롬프트 엔지니어링 개선 아이디어
- 한국 AI 기본법 외 다른 법률 자동화 니즈

### 버전 2: 문제 해결 중심

**제목**: Show GN: 법무법인 2,000만원짜리 AI 기본법 컨설팅을 자동화했습니다

**본문**:
AI 기본법 시행 후 AI 스타트업들이 공통적으로 겪는 문제가 있습니다:
- 고영향 AI 해당 여부 판정에 법률 전문가 필요 (시간 2~4주, 비용 2,000만원+)
- 영향평가서 작성에 15개 항목 × 최소 500자 = 7,500자 이상 필요
- 법 조문 55개 + 시행령 + 가이드라인 읽고 해석하는 것만 수주일

스타트업얼라이언스 조사 결과 98%가 대응 체계를 갖추지 못한 이유입니다.

**해결책: GPT-4o로 자동화**

AILEX는 AI 기본법 전용 컴플라이언스 도구입니다:
1. AI 시스템 정보 입력 (서비스명, 목적, 사용자 규모, 데이터 유형 등)
2. GPT-4o가 11개 영역별 고영향 기준 분석
3. 5분 만에 판정 결과 + 근거 법 조문 제공
4. 고영향 판정 시 영향평가서 자동 생성 (30분)

**기술 스택**:
- Frontend: Next.js 15 + shadcn/ui
- Backend: Supabase (Auth + PostgreSQL + Storage)
- AI: GPT-4o API (Prompt Caching 활용)
- 문서 생성: docx 라이브러리 + Puppeteer (PDF)

**비용 구조**:
- 기존: 법무법인 컨설팅 건당 2,000만원
- AILEX: 무료 자가진단 + Pro 월 30만원 (1/67 비용)

**한계점**:
- 복잡한 케이스(다수 AI 시스템 통합, 글로벌 규제 동시 대응 등)는 전문가 필요
- 계도기간 중이라 실제 과태료 부과 사례가 없어 리스크 검증 어려움
- 법 개정 시 프롬프트 업데이트 필요

현재 베타 테스터 모집 중입니다. AI 기본법 대응이 필요하신 분들의 피드백 환영합니다.

## 댓글 응답 전략

### 기술 질문 대응

**Q: 프롬프트는 어떻게 설계했나요?**
A: 3단계 구조입니다:
1. System Prompt: AI 기본법 시행령 전문 + "법률 분석가" 역할 부여
2. Few-shot Examples: 과기부 가이드라인의 30개 예시 (고영향/일반 판정 포함)
3. User Input: 기업의 AI 시스템 정보 (JSON 형태)

핵심은 "불확실성 처리"입니다. 확실한 판정이 어려울 때 "모호함" 응답을 반환하도록 설계했고, 이 경우 전문가 검토를 권장합니다.

판정 로직 일부를 GitHub에 공개할 예정이니 코드 리뷰 부탁드립니다.

**Q: Prompt Caching을 어떻게 활용했나요?**
A: GPT-4o의 Prompt Caching 기능으로 시스템 프롬프트(AI 기본법 전문, 약 10K tokens)를 캐싱했습니다.
- 첫 API 호출: $3/1M tokens (일반 요금)
- 이후 호출: $0.30/1M tokens (90% 할인)
판정 1건당 비용이 $0.30 → $0.03으로 감소했습니다.

캐시는 5분간 유지되므로, 동일 사용자의 반복 판정이나 배치 처리에 효과적입니다.

**Q: Next.js 15의 어떤 기능을 활용했나요?**
A: App Router + Server Actions를 핵심으로 사용했습니다:
- Server Actions로 GPT-4o API 호출 (환경변수 보안)
- Streaming으로 판정 진행 상황 실시간 표시
- Partial Prerendering(PPR)로 로딩 성능 개선
- Middleware로 인증 체크 & RLS 정책 적용

Vercel에 배포했고, Edge Runtime으로 한국 사용자에게 빠른 응답 제공합니다.

### 법률/비즈니스 질문 대응

**Q: AI 판정이 틀리면 법적 책임은 누가 지나요?**
A: 중요한 질문입니다. AILEX의 모든 판정에는 법적 면책 고지가 포함됩니다:
"본 판정은 참고용이며, 최종 확인은 법률 전문가에게 받으시기 바랍니다."

AILEX는 "초기 진단 도구"로 포지셔닝하며, 복잡한 케이스는 법무법인 검토를 권장합니다.
실제 법적 책임은 기업이 지게 되므로, AI 도구는 리스크 판단을 돕는 보조 수단입니다.

**Q: 법무법인이 이걸 쓸 이유가 있나요?**
A: 법무법인의 업무 효율화 도구로 활용 가능합니다:
- 초기 진단 자동화로 컨설팅 시간 단축
- 표준 문서 초안 생성 후 법률 검토
- 소액 건(스타트업 대상)을 AILEX로 처리하고 수익 분배

실제로 중소 법무법인 파트너십을 논의 중이며, "AILEX + 전문가 검토" 패키지를 고려하고 있습니다.

**Q: 오픈소스로 공개할 계획은?**
A: 핵심 엔진은 프로프라이어터리지만, 다음을 공개할 예정입니다:
- 고영향 판정 로직 (프롬프트 + 처리 플로우)
- AI 기본법 조문 구조화 데이터 (JSON)
- 판정 정확도 테스트 스크립트

커뮤니티 피드백으로 정확도를 계속 개선하고 싶습니다.
